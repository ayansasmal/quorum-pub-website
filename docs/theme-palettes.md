# Quorum — Theme Palettes

Four palettes from the hifi design handoff. **Library Slate** is the current active theme.
All four are ready to drop into `globals.css` for a future theme switcher.

Each palette defines light + dark mode tokens mapped to the same CSS variable names — swapping
themes is a `:root` variable swap, nothing else.

## Implementing a Theme Switcher

All four palettes share identical CSS variable names. The swap is a single block swap on `:root`.

### Option A — CSS data attribute (recommended)

```css
/* globals.css — replace the current :root blocks */

:root[data-theme="library-slate"]        { /* ... library slate light tokens ... */ }
:root[data-theme="library-slate"][data-mode="dark"]   { /* ... library slate dark tokens ... */ }

:root[data-theme="warm-editorial"]       { /* ... warm editorial light ... */ }
:root[data-theme="warm-editorial"][data-mode="dark"]  { /* ... warm editorial dark ... */ }

/* etc. */

/* System dark mode fallback when no data-mode is set */
@media (prefers-color-scheme: dark) {
  :root:not([data-mode]) { /* active palette dark tokens */ }
}
```

```tsx
// ThemeSwitcher client component
'use client'
import { useEffect, useState } from 'react'

export function ThemeSwitcher() {
  const [theme, setTheme] = useState('library-slate')
  useEffect(() => {
    document.documentElement.dataset.theme = theme
  }, [theme])
  // render picker...
}
```

### Option B — CSS custom property override (simpler, one palette)

Each `:root` block in `globals.css` already uses unique variable names. To add a second
theme, add a second `[data-theme="X"]` block and flip `document.documentElement.dataset.theme`.

### Variable name mapping (handoff → `globals.css`)

The hifi uses camelCase (`primarySoft`, `bgAlt`); `globals.css` uses kebab-case (`--primary-soft`, `--bg-alt`).
The `paletteToVars()` function in `project/hifi/theme.jsx` handles this conversion automatically.
